# OfficeDeskScene
Office desk scene created for my CS-330: Computer Graphics and Visualization class

1. How do I approach designing software?

Working on this project helped me improve my software design skills by teaching me how to break a large project into smaller, more manageable pieces. Instead of trying to build the entire scene at once, I focused on creating one object at a time before adding textures, materials, lighting, and camera controls. This made it much easier to identify and fix problems as they came up.

Throughout the project, I followed an iterative design process. I started with a rough layout of the scene using basic shapes, then gradually improved it by adding more detail and realism. As the milestones progressed, I refined the placement of objects, experimented with textures and lighting, and adjusted the overall appearance until I was satisfied with the final result.

2. How could tactics from your design approach be applied in future work?

This project reinforced the importance of developing software in small, incremental steps. Rather than trying to complete everything at once, I focused on getting one feature working before moving on to the next. This approach made debugging much easier and helped me avoid introducing multiple problems at the same time.

One of the biggest development strategies I used was constant testing and iteration. Almost every change required compiling and running the program to verify that it produced the expected result. This was especially true when adjusting object positions, textures, lighting, and material properties. Through each milestone, my approach became more organized, and I relied more on reusable functions and modular code instead of placing everything inside a single function. By the end of the project, my code was much cleaner, easier to read, and easier to maintain.

3. How can computer science help me in reaching my goals?

Although I do not plan to specialize in computer graphics, this project helped strengthen many programming skills that apply to all areas of software development. I gained additional experience with C++, debugging, problem solving, and organizing larger projects into reusable components. I also developed a better understanding of how graphics pipelines, transformations, textures, materials, and lighting all work together to create interactive applications.

These skills will benefit me throughout the rest of my computer science education and in my professional career. My long-term goal is to become a software engineer, and this project reinforced the importance of writing modular code, testing changes frequently, and approaching complex problems one step at a time. While I may not build 3D graphics applications professionally, the programming practices and problem-solving techniques I developed during this course will carry over into many other areas of software development.
